⚡ 为什么选 Pro Micro 3.3V / 8MHz？

优点说明
电压完美匹配SX1278 是 3.3V 器件。3.3V Pro Micro 的 IO 也是 3.3V，无需任何电平转换。
18650 直供18650 满电 4.2V → 3.7V 标称，可直接接 Pro Micro 3.3V 的 RAW 引脚（内置 LDO 稳压）。一块电池直驱，无需升压。
功耗更低8MHz (vs 16MHz) + 3.3V (vs 5V) → 空载电流约 5mA vs 20mA，续航翻倍。
USB HID 正常ATmega32U4 在 3.3V/8MHz 下原生支持 USB，Keyboard.h 库完全可用。

⚠️ 烧录注意事项

SX1278 的 SPI 引脚 (D14/D15/D16) 也是 Pro Micro 的 ISP 烧录引脚。烧录固件前先把 SX1278 的 SCK/MOSI/MISO 拔掉，烧完再接上，否则会冲突。

💻 软件方案

3.1 开发环境准备

1安装 Arduino IDE：Arduino IDE 2.x
2添加开发板：文件 → 首选项 → 附加开发板管理器网址，添加 https://raw.githubusercontent.com/sparkfun/Arduino_Boards/main/IDE_Board_Manager/package_sparkfun_index.json
3安装开发板支持：工具 → 开发板 → 开发板管理器 → 搜索 SparkFun AVR Boards → 安装
4选择开发板：工具 → 开发板 → SparkFun Pro Micro → 处理器选 ATmega32U4 (3.3V, 8MHz)
5安装 RadioLib 库：项目 → 加载库 → 管理库 → 搜索 RadioLib by Jan Gromes → 安装

3.2 发射端 (TX) — 固件代码

// ==========================================
// 发射端 (TX) — Pro Micro 3.3V + SX1278 LoRa
// 按键 → LoRa 发送 "UP" / "DOWN"
// ==========================================

#include <RadioLib.h>

// SX1278 SPI 引脚 (Pro Micro)
#define LORA_NSS   10
#define LORA_DIO0  2
#define LORA_RST   9

// 按键引脚
#define BTN_UP     4
#define BTN_DOWN   5

SX1278 radio = new Module(LORA_NSS, LORA_DIO0, LORA_RST);

void setup() {
  pinMode(BTN_UP,   INPUT_PULLUP);
  pinMode(BTN_DOWN, INPUT_PULLUP);

  // 初始化 LoRa: 433.0 MHz, SF12, BW=125kHz, CR=4/5
  int state = radio.begin(433.0, 125.0, 12, 5, 0x12);
  if (state != RADIOLIB_ERR_NONE) {
    // 初始化失败，死循环 (调试时改为 Serial.print 查看错误)
    while (true);
  }

  radio.setOutputPower(17);  // 发射功率 17dBm (最大)
}

void loop() {
  static unsigned long lastSend = 0;
  unsigned long now = millis();

  if (now - lastSend > 200) {
    if (digitalRead(BTN_UP) == LOW) {
      int state = radio.transmit("UP");
      lastSend = now;
    }
    if (digitalRead(BTN_DOWN) == LOW) {
      int state = radio.transmit("DOWN");
      lastSend = now;
    }
  }

  delay(20);
}

3.3 接收端 (RX) — 固件代码

// ==========================================
// 接收端 (RX) — Pro Micro 3.3V + SX1278 LoRa
// LoRa 接收 → USB HID 模拟 Page Up / Page Down
// ==========================================

#include <Keyboard.h>
#include <RadioLib.h>

// SX1278 SPI 引脚
#define LORA_NSS   10
#define LORA_DIO0  2
#define LORA_RST   9
#define LED_PIN    4

SX1278 radio = new Module(LORA_NSS, LORA_DIO0, LORA_RST);

void setup() {
  pinMode(LED_PIN, OUTPUT);

  // 初始化 LoRa: 参数必须与发射端完全一致
  int state = radio.begin(433.0, 125.0, 12, 5, 0x12);
  if (state != RADIOLIB_ERR_NONE) {
    while (true);
  }

  // 启动持续接收模式
  radio.startReceive();

  Keyboard.begin();

  // 启动信号: LED 闪烁 3 次
  for (int i = 0; i < 3; i++) {
    digitalWrite(LED_PIN, HIGH);
    delay(150);
    digitalWrite(LED_PIN, LOW);
    delay(150);
  }
}

void loop() {
  String msg;
  int state = radio.receive(msg);

  if (state == RADIOLIB_ERR_NONE) {
    digitalWrite(LED_PIN, HIGH);

    if (msg == "UP") {
      Keyboard.press(KEY_PAGE_UP);
      delay(50);
      Keyboard.releaseAll();
    }
    else if (msg == "DOWN") {
      Keyboard.press(KEY_PAGE_DOWN);
      delay(50);
      Keyboard.releaseAll();
    }

    digitalWrite(LED_PIN, LOW);
  }

  // 重新进入接收模式
  radio.startReceive();
}

📝 完整开发步骤

Part A：发射端 (遥控器)

1搭建 Arduino 开发环境
安装 Arduino IDE → 添加 SparkFun 板支持 → 选择 Pro Micro 3.3V/8MHz → 安装 RadioLib 库。
2SX1278 硬件接线
按上方接线图连接 SX1278 (VCC→3.3V, GND→GND, NSS→D10, SCK→D15, MOSI→D16, MISO→D14, DIO0→D2, RST→D9)。焊接弹簧天线到 SX1278 的 ANT 焊盘。
3按键接线
两个按键各一端接 GND，另一端分别接 D4 (Page Up) 和 D5 (Page Down)。利用芯片内部上拉 (INPUT_PULLUP)，无需外部电阻。
4供电方案
18650 电池 (3.7-4.2V) → Pro Micro RAW 引脚。Pro Micro 3.3V 版内置 MIC5219 LDO，输入 2.5-12V 稳压到 3.3V。一块电池直驱，非常简洁。
5烧录固件
⚠️ 先拔掉 SX1278 的 SCK/MOSI/MISO (SPI 与 ISP 冲突) → 编译上传发射端代码 → 接回 SPI 线。
6测试发射端
接上 USB 用串口监视器(9600bps)查看，或与接收端联调。按下按键，SX1278 的 DIO0 会短暂拉高表示发送完成。

Part B：接收端 (接收器)

1硬件接线
与发射端完全相同的 SPI 接线方式，另外 D4 接 LED+220Ω→GND。
2烧录固件
拔掉 SPI 线 → 上传接收端代码 → 接回。选择开发板 "SparkFun Pro Micro (3.3V, 8MHz)"。
3插入电脑验证 USB HID
LED 闪烁 3 次表示就绪 → 电脑应识别出 USB 键盘 → 按下遥控器 → Page Up/Down 生效。
4拉距测试
先在 5m 内调通 → 逐步拉远 → 测试 100m、200m、300m。LoRa 的优势在此时显现。

⚙️ LoRa 参数配置详解

参数代码表示本方案取值说 明
频率433.0 (MHz)433.0中国 ISM 免许可频段。两边必须一致。
带宽 BW125.0 (kHz)125kHz越小距离越远。125k 是最佳平衡点。
扩频因子 SF12SF126~12，越大越远但越慢。SF12=最远距离。
编码率 CR5 (即 4/5)4/5纠错编码率。5=4/5, 8=4/8。越小抗干扰越强。

⚠️ 发射端和接收端的频率、BW、SF、CR、Sync 必须完全一致！

🔧 关键技术要点

1. 为什么 Pro Micro 3.3V 的 USB 仍然可用？

ATmega32U4 的 USB 控制器有自己的 3.3V 稳压器和独立的 48MHz 时钟（从内部 PLL 产生），不依赖主 CPU 时钟和 IO 电压。因此即使在 3.3V/8MHz 下运行，USB 仍然正常工作。无数 Pro Micro 3.3V 项目已验证。

2. RadioLib 库的接收模式

RadioLib 的 receive() 方法是阻塞式的。配合 DIO0 中断，SX1278 在收到数据后拉高 DIO0，RadioLib 自动退出阻塞。对于翻页遥控器的低频率按键场景，这个开销可以忽略。

3. 抗干扰与多设备并存

  • 同步字 Sync Word：0x12 (可改 0x00-0xFF) 作为网络 ID，不同 Sync Word 的 LoRa 设备互不干扰。
  • LoRa 天然优势：扩频通信同一频点可同时存在多路信号而不冲突。
  • 如果信号冲突：换 Sync Word 或换频道 (如 434.0MHz)。

4. 低功耗优化

  • Pro Micro 3.3V/8MHz 空载 ~5mA
  • 利用 Arduino 的 <avr/sleep.h> 进入 IDLE 睡眠，按键中断唤醒
  • SX1278 的 sleep() 功耗仅 0.2μA
  • 2000mAh 电池理论可用数周至数月

5. 天线注意事项

  • SX1278 模块通常附带弹簧天线，焊接到 ANT 焊盘
  • 天线应远离金属物体，不被外壳屏蔽
  • 如需更好距离，可换 433MHz 棒状天线 (SMA 接口需另买转接板)
  • 天线方向影响很大：TX 和 RX 天线尽量平行放置
